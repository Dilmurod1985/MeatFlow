# MeatFlow MVP

Система управления производством для мясоперерабатывающего предприятия.

## Описание

MeatFlow - это веб-приложение для управления производственными процессами на мясоперерабатывающем предприятии. Система позволяет отслеживать остатки сырья, планировать производство и мониторить загрузку цехов.

## Функционал

- **Управление цехами**: 5 стандартных цехов (Филе, Котлеты, Разделка, Фарш, Шпажирование)
- **Приемка сырья**: Учет поступающего сырья с указанием веса и названия
- **Планирование производства**: Установка планов на каждый день
- **Мониторинг загрузки**: Отслеживание процента выполнения плана в реальном времени
- **Выпуск готовой продукции**: Фиксация произведенной продукции с автоматическим списанием сырья
- **Дашборд**: Центральная панель управления с обзором всех цехов

## Технологии

- **Backend**: FastAPI (Python)
- **Frontend**: HTML/CSS/JavaScript
- **База данных**: SQLite (по умолчанию), поддерживает PostgreSQL
- **ORM**: SQLAlchemy

## Установка и запуск

### Требования

- Python 3.8+
- pip

### Быстрый старт

1. Клонировать репозиторий:
```bash
git clone <repository-url>
cd MeatFlow
```

2. Установить зависимости:
```bash
pip install fastapi uvicorn sqlalchemy python-multipart
```

3. Запустить сервер:
```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

4. Открыть в браузере:
- Дашборд: `http://localhost:8000/`
- Панель цехов: `http://localhost:8000/panel`

## Конфигурация

### Переменные окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Основные переменные:
- `DATABASE_URL` - URL базы данных (по умолчанию: `sqlite:///./meatflow.db`)

### Пример конфигурации для PostgreSQL

```
DATABASE_URL=postgresql+psycopg2://user:password@localhost:5432/meatflow
```

## API Эндпоинты

- `GET /` - Дашборд
- `GET /panel` - Панель управления цехами
- `POST /receive` - Приемка сырья
- `GET /stock/{workshop_id}` - Получение статуса цеха
- `POST /set-plan` - Установка плана производства
- `POST /release/{workshop_id}` - Выпуск готовой продукции
- `GET /status_all` - Статус всех цехов

## Использование

1. **Планирование**: На дашборде установите план для каждого цеха
2. **Приемка**: Нажмите "Приемка" для внесения сырья в цех
3. **Производство**: Отслеживайте загрузку цехов в реальном времени
4. **Выпуск**: Нажмите "Выпуск" для фиксации готовой продукции

## Разработка

### Запуск в режиме разработки

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### База данных

База данных SQLite создается автоматически при первом запуске. Для использования PostgreSQL укажите соответствующий `DATABASE_URL`.
